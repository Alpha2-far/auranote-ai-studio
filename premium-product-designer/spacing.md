# Philosophy

Spacing is not empty space.

Spacing is structure.

Spacing is hierarchy.

Spacing is rhythm.

Spacing is silence.

Interfaces become premium because of spacing,
not because of effects.

---

# Core Principle

Never ask

"How much space should I add?"

Ask

"What relationship should these elements communicate?"

Spacing communicates relationship.

---

# Law 1

Closer means related.

Farther means different.

Nothing else.

---

# Law 2

Equal relationships

↓

Equal spacing.

Never use different spacing
for identical relationships.

---

# Law 3

Spacing creates hierarchy.

Increase spacing

↓

Increase perceived importance.

Reduce spacing

↓

Increase perceived connection.

# Mobile

Reduce spacing gradually.

Never remove spacing completely.

Hierarchy must survive
smaller screens.

---

# Anti Patterns

Reject

Random spacing.

Different spacing
for identical elements.

Nested padding.

Padding inside padding.

Crowded cards.

Buttons touching borders.

Multiple unrelated spacing systems.

---

# Premium Test

Ask

Can I remove one element
instead of reducing spacing?

If YES

Remove the element.

Never sacrifice breathing room
to keep unnecessary UI.

---

# Review Questions

Does spacing communicate hierarchy?

Can relationships be understood
without reading?

Is rhythm consistent?

Does every section breathe?

Does whitespace improve understanding?

Can any spacing value
be replaced by the system scale?

# Final Principle

Spacing is invisible.

When users notice spacing,

it is usually wrong.
