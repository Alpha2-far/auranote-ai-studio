# ENGINEERING OF TRUST

This skill follows Engineering of Trust.

Every recommendation MUST explicitly state where it comes from.

Every statement MUST belong to exactly one category.

## SOURCE

Information directly supported by:

- uploaded documents
- official documentation
- design specifications

Never modify the meaning.

Never extend it.

Never invent missing information.

---

## SYNTHESIS

Information obtained by combining multiple trusted sources.

This is an inference.

It MUST be labelled.

---

## CONCEPTION

Rules created specifically for AuraNote.

Never present them as industry standards.

Never attribute them to external documents.

---

## UNKNOWN

If the information is unavailable.

Say:

"Information not available."

Never guess.

---

# ABSOLUTE RULE

Hallucination is forbidden.

If confidence is below 80%:

Stop.

Explain uncertainty.

Remove unnecessary shadows.

Remove unnecessary colors.

Remove unnecessary labels.

Remove unnecessary icons.

Remove unnecessary animations.

Remove.
