Purpose

Define how complete pages are designed.

A page is not a collection of components.

A page is a user journey.

---

# Philosophy

Every page answers one question.

Never two.

Never three.

One page

↓

One primary objective.

Examples

Editor

↓

Write.

Search

↓

Find.

Settings

↓

Configure.

Dashboard

↓

Understand.

Never mix purposes.

---

# Universal Page Structure

Every page follows

Header

↓

Primary Content

↓

Supporting Context

↓

Actions

↓

Footer (optional)

Users should always know

Where they are.

What they can do.

What happens next.

---

# Reading Order

Users should naturally scan

1

Identity

↓

2

Context

↓

3

Primary Content

↓

4

Primary Action

↓

5

Secondary Information

Never interrupt this flow.

---

# Header

Purpose

Orientation.

Not decoration.

Contains

Page Title

Subtitle (optional)

Primary Action (optional)

Search (optional)

Never overload headers.

---

# Sidebar

Purpose

Persistent navigation.

Never temporary actions.

Rules

Stable.

Predictable.

Always recognizable.

Never change order automatically.

---

# Workspace

Purpose

The user's work.

Everything else supports it.

The workspace receives

the most space.

Never advertisements.

Never decorative panels.

Never unrelated widgets.

---

# Inspector Panel

Purpose

Contextual information.

Not primary work.

Appears only

when useful.

Can disappear.

Must never hide

critical functionality.

---

# Empty State

Purpose

Teach.

Never apologize.

Every empty state answers

What happened?

Why?

What now?

Never display

"No Data"

without explanation.

---

# Loading State

Purpose

Maintain confidence.

Users should understand

that work is progressing.

Prefer

Skeletons.

Avoid

Large spinners.

---

# Error State

Purpose

Recovery.

Never punishment.

Every error explains

What happened.

Why.

How to recover.

Never

Something went wrong.

---

# Success State

Purpose

Reassurance.

Communicate

Completion.

Not celebration.

Subtle.

Professional.

---

# Search Page

Primary focus

Search.

Everything else disappears.

Search should dominate

the viewport.

Results should appear

immediately below.

---

# Editor Page

Classification

CONCEPTION

Confidence

100%

Purpose

Thinking.

Writing.

Understanding.

Rules

Editor occupies

maximum available space.

Navigation becomes secondary.

Distractions disappear.

Whitespace increases.

Reading comfort first.

---

# Dashboard

Purpose

Decision making.

Never decoration.

Dashboards answer

What changed?

What matters?

What requires attention?

Never

Display data

without interpretation.

---

# Settings

Purpose

Configuration.

Never exploration.

Rules

Group related settings.

One concern

per section.

Use plain language.

Avoid technical jargon.

---

# AI Chat Page

Purpose

Conversation.

Rules

Messages remain readable.

AI responses appear progressively.

Code blocks preserve formatting.

Citations remain visible.

Reasoning is separated from answers.

Never overload one message.

---

# Documentation Page

Purpose

Learning.

Rules

Wide reading column.

Excellent typography.

Strong hierarchy.

Persistent table of contents.

Never narrow text unnecessarily.

---

# Detail Page

Purpose

Deep understanding.

Rules

Overview first.

Details second.

Metadata last.

Never expose raw complexity first.

---

# Mobile

Hierarchy survives.

Spacing adapts.

Navigation simplifies.

Content remains primary.

Never remove functionality

only because of screen size.

---

# Desktop

Use available space

to improve clarity.

Never fill empty space

without purpose.

---

# Progressive Disclosure

Reveal complexity gradually.

Beginners

see simplicity.

Experts

discover power.

Never overwhelm

new users.

---

# Context Preservation

Navigation

must never destroy context.

Users should always know

where they came from.

where they are.

where they can go.

---

# AI First Pages

AuraNote is AI-native.

Every page should answer

Can AI reduce effort here?

If YES

Integrate AI naturally.

Never force AI

where it adds no value.

---

# Cognitive Load

Every page must minimize

Decision Count

Visual Noise

Reading Effort

Memory Load

Task Switching

The user should think

about their work.

Never about the interface.

---

# Premium Review

Ask

Can users understand this page

within five seconds?

Can they locate

the primary action

within two seconds?

Can they complete

their task

without confusion?

Can one section

be removed?

If YES

Remove it.

---

# Rejection Conditions

Reject the page if

Multiple competing objectives exist.

Primary content is unclear.

Navigation changes unexpectedly.

Whitespace disappears.

AI interrupts workflows.

Actions compete.

Visual noise increases.

The page feels crowded.

---

# Final Principle

A great page

is not remembered

because it was beautiful.

It is remembered

because users forgot

it was there.

They only remember

what they accomplished.
